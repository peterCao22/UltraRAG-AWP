# 中文注释模板与示例（可复制）

本文件提供“直接复制即可用”的中文注释模板。你可以按项目情况精简，但不要删除“参数/返回”这两个必备段落。

---

## 1) 模块级注释模板（文件头）

```python
"""
一句话：本文件做什么。

关键点：
- 关键点1：解释核心功能/流程（不要复述代码）
- 关键点2：解释数据口径/约束（日期、频率、复权、交易日等）
- 关键点3：解释性能/资源策略（批量、缓存、分批、限速）

输入：
- 输入来源/文件/表：xxx
- 关键字段/格式：xxx

输出：
- 输出文件/产物：xxx
- 输出字段/口径：xxx

运行方式：
- python path/to/file.py [args...]
- 或：调用入口函数 main()/run()

注意：
- 风险点1：例如“避免未来函数/数据泄漏”
- 风险点2：例如“接口配额/超时/重试”
"""
```

## 2) 函数/方法 docstring 模板（必含：参数/返回）

```python
def func(a, b, *, mode="fast"):
    """
    作用：一句话说明这段函数的目的（强调“为什么要做”）。

    参数：
    - a：含义、单位/格式、允许范围、默认值语义（如有）
    - b：含义、单位/格式、允许范围、默认值语义（如有）
    - mode：可选值与语义；说明什么时候用哪个选项

    返回：
    - 返回值：类型/结构（DataFrame/Dict/tuple），关键字段含义，空结果时返回什么

    异常（按需）：
    - ValueError：什么情况下会抛出
    - RuntimeError：外部依赖失败时可能抛出

    示例（按需）：
    - 最小示例（只展示关键参数与关键输出）
    """
```

> 提醒：不强制补类型标注，但如果你能准确写出 `-> pd.DataFrame` 这类返回类型，会显著提高可读性。

## 3) 类 docstring 模板

```python
class Foo:
    """
    职责：一句话说明这个类负责什么、不负责什么（边界清晰）。

    关键属性：
    - self.x：含义、单位/口径、生命周期
    - self.y：含义、是否可为 None

    关键方法：
    - method1：做什么、输入输出、注意事项
    - method2：做什么、输入输出、注意事项

    示例（按需）：
    - foo = Foo(...)
      foo.method1(...)
    """
```

## 4) 关键逻辑行内注释示例（“为什么”优先）

### 4.1 去重/覆盖语义

```python
merged = pd.concat([old_df, new_df], ignore_index=True)

# 用 (date, instrument) 去重：同一股票同一天只保留一条记录。
# keep='last' 的语义是“新数据覆盖旧数据”（用于修复历史缺字段/缺行）。
merged = merged.drop_duplicates(subset=["date", "instrument"], keep="last")
```

### 4.2 日期口径统一（避免隐性重复）

```python
# 统一日期到 YYYY-MM-DD，避免 'YYYY-MM-DD 00:00:00' 与 'YYYY-MM-DD' 造成同日重复。
df["date"] = df["date"].astype(str).str.slice(0, 10)
```

### 4.3 性能/配额策略

```python
# 分批查询：避免一次 IN 过大导致单次响应过大、或触发接口限制。
for i in range(0, len(instruments), batch_size):
    batch = instruments[i:i + batch_size]
    # 限速：避免请求过快触发服务端限流/不稳定（502/unavailable）。
    time.sleep(0.5)
```

## 5) 常用中文术语建议（统一口径）

- 配额 / 流量：统一使用其中一个（建议“配额”）
- 增量更新：只拉取缺失新数据
- 全量更新：重新拉取完整区间（成本高）
- 落盘：写入本地文件（CSV/Parquet/DB）
- 缓存：短期复用，避免重复查询
- 口径：字段含义/统计方式/复权方式/时间对齐方式
