## 0. 响应式与终端（手机自适应）

### 0.1 断点约定（与 `02_界面设计规范.md` 一致）

| 名称 | 视口宽度 | 说明 |
|------|-----------|------|
| Desktop | ≥1024px | Nous 式：侧栏常显 + 主区 + Composer |
| Tablet | 768px～1023px | 对话侧栏可改为**默认收起 + 顶栏汉堡**打开抽屉 |
| Mobile | &lt;768px | **对话**：侧栏**抽屉/遮罩**（默认关），主区全宽；**管理**：侧栏抽屉 + 内容区单列 |

自测建议：至少抽测 **390×844**（常见手机逻辑宽）与 **375×667**（偏小屏）。

### 0.2 对话页（手机必达）

- HTML 必须包含 `<meta name="viewport" content="width=device-width, initial-scale=1">`（或等价写法）。
- **侧栏**：窄屏默认不占主区宽度；通过**汉堡按钮**（置于主区左上角或顶缘）打开/关闭；打开时**半透明遮罩**点击可关。
- **Workspace**：消息列表区域 `flex:1` + `min-height:0`，避免与底栏 Composer 抢高度导致滚动异常。
- **Composer**：贴底；考虑 `padding-bottom: env(safe-area-inset-bottom)`，避免被 iOS 底部横条遮挡。
- **气泡宽度**：窄屏可适当放宽 `max-width`（如 88%～92%），保证可读。
- **来源与 Lightbox**：缩略图横滑区域 `overflow-x:auto`；Lightbox 在小屏上全宽展示，关闭按钮触控区域足够大。

### 0.3 管理后台（手机可读可操作为主）

- **&lt;768px**：左侧导航收入**抽屉**；Header 保留标题 + 汉堡 +「返回对话」。
- **表格**：优先允许**横向滚动**（`overflow-x:auto` + `min-width`）；进阶可做「小屏卡片列表」与表格二选一（Sprint 3/4 按工时取舍）。
- **按钮与链接**：触控目标建议 **≥44×44px**（或等价可点区域）。
- **Modal**：小屏下接近全屏或宽度 `min(100vw-32px, 480px)`，避免超出视口。

### 0.4 与桌面稿的关系

Figma 中除 **1440 宽** 画板外，建议增加 **`CH-mob`（390 宽）**、**`AD-mob`（390 宽）** 各一张，避免实现阶段临时挤布局。

## 1. 对话主页（`/chat`）

> **布局定稿（Nous 式）**：参考 [Nous Chat](https://chat.nousresearch.com/) 一类产品的信息架构——**左侧窄栏**（品牌、操作、选库、入口）+ **右侧主工作区**（仅消息流与引用）+ **主区底栏**（多行输入与发送）。**不设**全宽顶栏承载选库；垂直空间优先给对话。

**侧栏信息架构（自上而下）：**

| 区块 | 内容 |
|------|------|
| 品牌区 | 产品名 / Logo（文案：AGV 知识库助手） |
| 主操作 | **新建对话** 按钮 |
| 选库 | **知识库** 下拉（`GET /api/kb`，过滤可用状态） |
| 可选 | **会话列表**（Sprint 1 可仅占位；多会话、localStorage 见后续迭代） |
| 底区 | **管理后台** 文字或图标链接 → `/admin` 或 `#/admin` |

### 1.2 交互细节

**知识库选择器：**
- 位于**左侧边栏**，顺序建议在「新建对话」下方（与品牌区的上下关系可在实现时微调，但须保持在侧栏内）
- 下拉列表显示所有 `status=ready`（及后端约定可用的状态）的知识库
- 选择后自动清空当前对话历史
- 无可用知识库时侧栏或主区显示「暂无知识库」，并附**管理后台**链接
- 记住上次选择（localStorage）

**智能体选择器（对话模式）：**
- 位于**主区底栏 Composer** 内、输入框**左侧**（或紧贴输入框上方的窄条，二选一以实现稿为准），样式参考 WeKnora：**「智能体 ▼」+ 当前项文案**（如「快速问答」「智能推理」）。
- 选项第一版：**快速问答**、**智能推理**；与后端字段 `agent_mode`：`quick` / `agent` 对应（详见 `04_API对接设计`）。
- **切换智能体不清空**当前消息列表；与「切换知识库清空历史」区分。
- 记住上次选择（localStorage，键名与前端约定一致即可）。
- 后端尚未实现真推理时：仍发 `agent_mode=agent`，依赖 SSE `meta.degraded` 提示「已降级为快速问答」一次；步骤区可无数据隐藏。

**消息输入框：**
- 多行 textarea，最大高度 120px，超出滚动
- Enter 发送，Shift+Enter 换行
- 流式输出期间禁用输入框和发送按钮（显示停止按钮）
- 字数限制 500 字，超限时边框变红

**流式输出：**
- AI 气泡先出现"思考中..."动画（三个点跳动）
- 收到第一个 chunk 后替换为实际文字，逐字追加
- Markdown 内容在流结束后一次性渲染（流中显示原始文本）
- 收到 `sources` 事件后，在气泡底部追加来源面板

**来源面板：**
- 默认折叠（显示"📄 引用来源（N 处）▾"）
- 点击展开，显示每个 source 的卡片
- 每张图片点击后弹出 Lightbox 放大查看
- 来源卡片摘要文字：最多显示 3 行，超出省略

**新建对话：**
- 位于**左侧边栏**主按钮区；点击清空 `messages` 数组
- 知识库下拉**保留当前选中**（与「切换知识库清空历史」区分）
- 输入框获取焦点（焦点落在主区 Composer）

**管理后台入口：**
- 位于**左侧边栏**底部（或侧栏与主区之间的弱分割线下方），始终可点进管理流程

**索引状态实时显示：**
- 文档上传后，状态变为 `indexing`，显示进度旋转图标
- 每 3 秒轮询 `GET /api/kb/<id>/status`
- 状态变为 `ready` 时停止轮询，刷新文档列表
- 若状态为 `error`，显示错误信息（红色文字）

**删除确认弹窗：**
```

[取消]  [确认删除]
```
