# Phase 3 界面设计规范

## 设计原则

1. **工业内网风格**：界面简洁、信息密度适中，适合操作人员在工厂内网环境使用
2. **功能优先**：避免过度装饰，以内容可读性为第一优先级
3. **响应提示清晰**：状态变化（加载、错误、成功）必须有明确视觉反馈
4. **低维护成本**：不依赖外网字体或 CDN，用系统字体栈

## 布局规范

### 对话页布局（Nous 式已定稿）

**结构**：`row` → 左 **Sidebar（260～280px）** + 右 **Workspace（flex:1, column）**；Workspace 内上为 **MessageList（flex:1, scroll）**，下为 **Composer（贴底，可选 max-height）**。无全宽顶栏。

```
┌──────────────┬─────────────────────────────────────────────┐
│ Sidebar      │ Workspace                                   │
│ · 品牌       │ · MessageList（可滚动）                      │
│ · 新建对话   │   · AI 左对齐 max-width ~72%                 │
│ · 知识库选择 │   · 用户右对齐                               │
│ · (会话列表) │   · 来源面板（AI 消息块内，可折叠）           │
│ · 管理后台链 │ · Composer：多行输入 + 发送                  │
└──────────────┴─────────────────────────────────────────────┘
```

侧栏背景可用 `--color-bg-sidebar`（见 Token）或浅灰，与主区 `--color-bg-base` 区分即可。

### 管理后台布局（侧边栏 + 内容区）

```
┌──────────────────────────────────────────────────────────┐
│  Header（高 56px）：Logo + "管理后台" + 返回对话页链接   │
├────────────┬─────────────────────────────────────────────┤
│  Sidebar   │  Content Area                               │
│  (220px)   │                                             │
│            │  面包屑 + 页面标题                          │
│  ▸ 知识库  │                                             │
│    管理    │  卡片列表 / 表格 / 表单                     │
│  ▸ 系统    │                                             │
│    状态    │                                             │
│            │                                             │
└────────────┴─────────────────────────────────────────────┘
```

## 组件规范

### 对话气泡

```
AI 气泡：
  背景 #ffffff，border 1px solid var(--color-border)
  圆角 var(--radius-lg)，左上角 0
  头像：机器人图标（SVG inline），蓝色背景圆形
  Markdown 内容区：line-height 1.8，table 带边框

用户气泡：
  背景 var(--color-primary)，文字白色
  圆角 var(--radius-lg)，右上角 0
  纯文本，无 Markdown 渲染
```

### 来源卡片（SourceCard）

```
边框：1px solid #e8f0fe，背景 #f8faff
圆角 var(--radius-md)
左侧 4px 蓝色边条（border-left: 4px solid var(--color-primary)）

内容：
  📄 [文档标题]（font-weight 600）
  段落摘要（灰色，2 行省略，font-size var(--text-sm)）
  图片行（水平滚动，缩略图 80×60px，cursor pointer）
```

### 状态徽章（StatusBadge）

```
building: 黄色背景 #fff7e6，黄色文字 #d46b08，旋转加载图标
ready:    绿色背景 #f6ffed，绿色文字 #389e0d，对勾图标
error:    红色背景 #fff2f0，红色文字 #cf1322，叹号图标
pending:  灰色背景 #f5f5f5，灰色文字 #8c8c8c，时钟图标
```

### 文件上传区（FileUpload）

```
正常态：
  虚线边框 2px dashed var(--color-border)，圆角 var(--radius-md)
  中心图标 + 文字"拖拽文件到此处或点击上传"
  支持格式说明（DOCX、PDF）

拖入态：
  边框变蓝 var(--color-primary)，背景 var(--color-primary-light)

上传中：
  线性进度条（填充色 var(--color-primary)）
  文件名 + 百分比显示
```

### Toast 通知

```
位置：右上角，距顶/右各 16px，固定定位
动画：从右侧滑入，3 秒后淡出

Success：绿色图标 + 消息文字
Error：  红色图标 + 消息文字（5 秒消失）
Info：   蓝色图标 + 消息文字
```

## 图标规范

不引入图标库（避免外网 CDN），使用内联 SVG：

```js
// 使用简单的 SVG 字符串常量
const ICONS = {
  robot: `<svg viewBox="0 0 24 24" fill="currentColor">...</svg>`,
  send: `<svg viewBox="0 0 24 24" fill="currentColor">...</svg>`,
  // ...
}
```

优先使用 Unicode 字符替代图标（如 📄 📷 ✅ ⚠️）

## 响应式断点（PC + 手机自适应）

| 断点 | 宽度 | 对话页（Nous） | 管理后台 |
|------|------|----------------|----------|
| Desktop | ≥1024px | 侧栏常显 + Workspace + Composer | 顶栏 + 侧栏 + 内容区 |
| Tablet | 768px～1023px | 侧栏可默认收起，汉堡打开抽屉 | 侧栏可收起为抽屉 |
| Mobile | &lt;768px | **侧栏抽屉 + 遮罩**；主区与 Composer 全宽可用 | **侧栏抽屉**；内容单列；表格横滑或卡片化 |

**触控**：主要操作控件建议最小点击区域 **44×44px**（或视觉上小但 `padding` 扩大热区）。

**安全区**：底栏 Composer、固定 Tab 等使用 `env(safe-area-inset-bottom)` 等避免刘海屏与 Home 指示条遮挡。

**自测宽度**：至少覆盖 **375px、390px** 逻辑宽度；横屏暂不强制单独稿，以不溢出、可滚动为底线。
