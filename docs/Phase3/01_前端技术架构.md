# Phase 3 前端技术架构

## 技术选型

### 核心框架：Vue 3（CDN 模式）

**选择理由：**
- 无需 Node/npm 构建环境，`<script src="vue.global.js">` 即可使用
- Composition API 适合逻辑复用（chat、kb 管理逻辑独立封装）
- 比纯 Vanilla JS 更易维护，比 React CDN 体积更小
- 内网可将 vue.global.js 下载到本地，无需外网访问

**版本**：Vue 3.4.x（ESM CDN 或 UMD CDN 均可，选 UMD 兼容性更好）

### 依赖库清单（全部本地化）

| 库 | 版本 | 用途 | 大小（gzip） |
|----|------|------|-------------|
| `vue.global.prod.js` | 3.4.x | UI 框架 | ~35KB |
| `marked.min.js` | 9.x | Markdown → HTML | ~20KB |
| `DOMPurify.min.js` | 3.x | XSS 过滤（配合 marked）| ~8KB |
| `highlight.min.js` | 11.x | 代码高亮（可选）| ~30KB |

> 不引入 UI 组件库（如 Element Plus），避免引入 1MB+ 的 CSS/JS。所有 UI 组件手写，与业务强绑定，体积可控。

## 架构设计

**对话主页布局（已定稿）**：采用 **Nous 式**（`03_页面功能设计.md` §1）——左 **Sidebar**（品牌、新建对话、知识库选择、可选会话列表、管理后台入口）+ 右 **Workspace**（`MessageList` + 底栏 `Composer`）；无全宽顶栏承载选库。

**响应式**：同一套页面需适配手机（`03` §0）。`index.html` 须含标准 `viewport` meta；窄屏下侧栏以 **抽屉 + 遮罩** 实现，避免与主内容并排挤压。样式以 **CSS 媒体查询 + 少量 Vue 状态**（如 `sidebarOpen`）为主，不引入移动端专用构建链。

### Vue 组件加载方式

使用 Vue 3 的 `defineComponent` + 原生 ES module（`.js` 文件），通过 `<script type="module">` 加载：

```html
<!-- index.html -->
<script src="vendor/vue.global.prod.js"></script>
<script type="module" src="main.js"></script>
```

```js
// main.js
import ChatMessage from './components/ChatMessage.js'
import SourceCard from './components/SourceCard.js'

const { createApp, ref, reactive } = Vue

const App = {
  components: { ChatMessage, SourceCard },
  // ...
}

createApp(App).mount('#app')
```

> 注意：IE 不支持 ES module，但内网环境已限定 Chrome 90+，无需 polyfill。

### Flask 静态文件路由

```python
# custom_app/app.py
from flask import send_from_directory

@app.route('/')
def index():
    return send_from_directory('frontend', 'index.html')

@app.route('/admin')
@app.route('/admin/')
def admin():
    return send_from_directory('frontend', 'admin.html')

@app.route('/login')
def login():
    return send_from_directory('frontend', 'login.html')

@app.route('/frontend/<path:filename>')
def frontend_static(filename):
    return send_from_directory('frontend', filename)
```

## 状态管理

不引入 Vuex/Pinia（CDN 环境配置复杂）。使用 Vue 3 `reactive()` + 模块级单例模式：

```js
// store/chat.js  
const { reactive, ref } = Vue

// 单例 store
export const chatStore = reactive({
  messages: [],
  currentKbId: null,
  isStreaming: false,
  
  addMessage(role, content) {
    this.messages.push({ role, content, sources: [], id: Date.now() })
  },
  
  setKb(kbId) {
    this.currentKbId = kbId
    this.messages = []  // 切换知识库时清空对话
  }
})
```

## 通信层设计

### SSE 流式接收（对话）

```js
// services/chatApi.js
export async function sendMessage(kbId, question, onChunk, onSources, onDone) {
  const response = await fetch('/api/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ kb_id: kbId, question, stream: true })
  })

  const reader = response.body.getReader()
  const decoder = new TextDecoder()
  
  while (true) {
    const { done, value } = await reader.read()
    if (done) { onDone(); break }
    
    const text = decoder.decode(value)
    // 解析 SSE 格式: "data: {...}\n\n"
    for (const line of text.split('\n')) {
      if (!line.startsWith('data: ')) continue
      const data = JSON.parse(line.slice(6))
      if (data.type === 'chunk') onChunk(data.content)
      if (data.type === 'sources') onSources(data.sources)
    }
  }
}
```

### REST API 封装

```js
// services/kbApi.js
const BASE = '/api/kb'

export const kbApi = {
  list: () => fetch(BASE).then(r => r.json()),
  create: (data) => fetch(BASE, { method: 'POST', ... }).then(r => r.json()),
  getDetail: (id) => fetch(`${BASE}/${id}`).then(r => r.json()),
  delete: (id) => fetch(`${BASE}/${id}`, { method: 'DELETE' }).then(r => r.json()),
  uploadDoc: (id, file) => {
    const form = new FormData()
    form.append('file', file)
    return fetch(`${BASE}/${id}/documents`, { method: 'POST', body: form }).then(r => r.json())
  },
  getStatus: (id) => fetch(`${BASE}/${id}/status`).then(r => r.json()),
}
```
