# Phase 2.5/3 能力增强路线（参考 WeKnora）

> 目标：在已落地的 Phase 2 基础版上，分阶段补齐平台化能力。  
> 原则：不破坏当前最小链路，按“可验证增量”推进。

## A. 多租户模型（优先级：高）

- 在核心表全面启用 `tenant_id`
- 默认租户 `default` 向显式租户迁移
- 约束：任一查询必须带租户上下文（后续与鉴权联动）

## B. 任务系统升级（优先级：高）

- 当前：单机轻量任务状态
- 升级方向：
  - 队列化执行
  - 并发上限
  - 重试与死信
  - 可取消任务

## C. 渠道化来源追踪（优先级：中）

- 统一 `channel` 字段，支持 `web/api/im/...`
- 知识条目、任务、对话日志均落库来源信息
- 便于审计与运营分析

## D. 安全与配置治理（优先级：中）

- API Key/敏感配置从明文文件逐步迁移至加密存储
- 增加配置变更审计日志
- 加强外部请求安全校验（如 SSRF 防护）

## E. 检索与存储后端可插拔（优先级：中）

- 保持 FAISS 默认路径
- 预留向量库后端切换能力
- 统一后端抽象，避免业务层感知具体实现

## F. Phase 2 到 Phase 4 的衔接

- Phase 2：多知识库 + 元数据 + 任务状态
- Phase 3：管理界面与流程可视化
- Phase 4：用户与权限、租户访问控制闭环
