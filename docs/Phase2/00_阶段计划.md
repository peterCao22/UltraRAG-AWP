# Phase 2 阶段计划（v1：基础落地 + 可扩展增强）

## 总体目标

基于当前已跑通的 Phase 1（文件型单库问答链路），新增多知识库管理能力，并为后续平台化能力预留扩展位：

1. 引入 SQLite 元数据管理（`db/app.sqlite`）
2. 提供知识库管理 API（创建、查询、更新、删除、状态查询）
3. 建立"文档入库/索引构建"任务状态流转
4. 保持 `/api/chat` 与 `/api/chat/markdown` 对现有 `agv_demo` 的兼容
5. 引入基于角色的访问控制（RBAC），用于企业内部多知识库权限管理

---

## 执行原则（先简后强）

- 先做"最小可用闭环"，不一次性引入全部平台特性
- Phase 2 专注后端 API，**前端管理界面推迟到 Phase 3**（参照 WeKnora 设计）
- 权限模型使用 **角色（Role）** 概念，而非多租户隔离：
  - 用户绑定角色，角色拥有对知识库的访问权限
  - 同一角色可访问多个知识库，同一知识库可被多个角色访问
- 任务流先实现单机串行/轻量异步，再演进为队列并发与限流
- 保持 Phase 1 问答链路零回归
