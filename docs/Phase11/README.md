# Phase 11 —— 生产化收尾（投产前的工程加固 + 用户体验扩展）

> **状态**：方向锚点（2026-05-16），详细计划待 Phase 10 完成后展开
> **前置**：[Phase 8](../Phase8/README.md) + [Phase 9](../Phase9/README.md) + [Phase 10](../Phase10/README.md) 全部完成
> **参考实现**：`D:\Peter2025\myCursor\WeKnora`（FAQ / 标签 / 意图理解 / 国际化）
> **定位**：从「内部 PoC」转为「**企业内部生产级 RAG 系统**」收尾阶段

---

## 一、阶段目标

Phase 1-10 完成后，custom_app 已具备：

- ✅ 强存储栈（Qdrant + Postgres + Neo4j）
- ✅ 检索算法（向量 + BM25 + Contextual + 可选 IRCoT）
- ✅ 图文联动（图片 caption + KG 桥接）
- ✅ 多租户 + KB 共享

但**还不能投产**——缺工程加固和用户体验。Phase 11 补齐这两块，让系统能**真正交付给业务**。

---

## 二、为什么拆 11.1 + 11.2

把 14 个收尾事项混在一个 Phase 里风险大（**8-10 周单 Phase 容易失控**），按"必须有 vs 体验好"分两轨：

| 子阶段 | 性质 | 优先级 | 工时 |
|--------|------|--------|------|
| [11.1 生产化必需](./PHASE_11_1_PLAN.md) | 不做就不能上线 | 🔴 必做 | 4-5 周 |
| [11.2 用户体验扩展](./PHASE_11_2_PLAN.md) | 做了用户更喜欢 | 🟡 重要 | 4-5 周 |

**最坏情况**：11.1 上线，11.2 边运营边迭代；不影响系统可用性。

## 三、Phase 11.1 主项（生产化必需）

| 子项 | 内容 | 工时 |
|------|------|------|
| 11.1.1 **结构化日志 + 归档** | JSON 日志 / 按类型分文件（app / audit / kg / ingest / chat）/ 按日滚动 + 7-30 天保留 | 2-3 天 |
| 11.1.2 **审计日志** | who-when-what：用户操作 + 问答记录 + KB 变更 | 3-4 天 |
| 11.1.3 **FAQ 库** | 独立向量空间；高置信度（≥0.85）直接命中不走 LLM | 1-1.5 周 |
| 11.1.4 **标签系统** | KB / Document 级标签；检索时按标签过滤范围 | 4-5 天 |
| 11.1.5 **Query 意图理解** | 闲聊 / 知识问答 / 数据查询 分流；不该走 RAG 的不走 | 3-4 天 |
| 11.1.6 **Rate Limiting + 配额计量** | 防单用户刷爆配额；每 tenant Gemini 调用统计 | 3-4 天 |

**详见** [PHASE_11_1_PLAN.md](./PHASE_11_1_PLAN.md)

## 五、本期**不**做的事

下列在前面讨论中被 explicitly 排除：

| 排除项 | 原因 | 推到哪 |
|--------|------|--------|
| Web 搜索能力 | 内网工厂场景合规未确认；闲聊用 Web 搜索性价比低 | 待业务确认后再议 |
| IM 接入（飞书 / Lark） | 工时 1.5 周；不如先把 Web 端做扎实 | 后续视真实需求 |
| Context Resolution（多轮指代消解） | 性质属"算法"非"产品扩展"；强依赖评测和长会话数据 | **Phase 12.1** |
| Docker 容器化 / k8s 部署 | 这些是 DevOps 工作，与本应用代码改造无关 | 由运维 / 部署 phase 独立处理 |
| HTTPS / 反向代理 / 密钥管理 | 同上，DevOps 范畴 | 同上 |
