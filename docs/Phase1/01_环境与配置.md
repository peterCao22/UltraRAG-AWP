## 1.4 检索模型配置（含 Google Embedding 适配说明）

### 为什么需要自定义适配层

`text-embedding-004` 已停用。当前可用的 Google 嵌入模型是 `gemini-embedding-001`（稳定版）和 `gemini-embedding-2-preview`（预览版）。

这两个模型的**原生 API 格式**与 OpenAI Embedding API 不同：

| 对比项 | OpenAI 格式 | Google 原生格式 |
|--------|------------|----------------|
| 接口路径 | `POST /v1/embeddings` | `POST /v1beta/models/{model}:embedContent` |
| 认证头 | `Authorization: Bearer <key>` | `x-goog-api-key: <key>` |
| 请求体 | `{"model": "...", "input": [...]}` | `{"model": "...", "content": {"parts": [{"text": "..."}]}}` |
| 返回体 | `data[].embedding` | `embeddings[].values` |

UltraRAG 的 `openai` backend 直接调 OpenAI SDK，**无法直接对接 Google 原生格式**。

**解决方案**：在 `custom_app/services/` 中编写 `google_embedder.py`，绕过 UltraRAG retriever 的 embed 步骤，直接调 Google 原生 API，产出同样格式的 `embedding.npy`。index 步骤仍用 UltraRAG FAISS，search 步骤仍用 UltraRAG retriever（search 时只需对查询文本做一次 embedding，同样走此适配层）。

### `servers/retriever/parameter.yaml` 配置

Retriever 配置改为**仅负责 index + search**，embed 步骤由自定义脚本完成：

```yaml
# servers/retriever/parameter.yaml
# Phase 1: embed 由 custom_app/services/google_embedder.py 完成
#          index / search 仍走 UltraRAG FAISS

# 模型名仅用于标识，实际 embed 不经过此 backend
model_name_or_path: gemini-embedding-001
backend: sentence_transformers      # 占位，index/search 不调用 embed 方法
backend_configs:
  sentence_transformers:
    trust_remote_code: true

index_backend: faiss
index_backend_configs:
  faiss:
    index_use_gpu: false
    index_chunk_size: 10000
    index_path: data/kb/agv_demo/index/index.index

corpus_path: data/kb/agv_demo/corpora/chunks.jsonl
embedding_path: data/kb/agv_demo/embedding/embedding.npy

batch_size: 16
top_k: 8
collection_name: agv_demo
```

### `custom_app/services/google_embedder.py`（新增文件）

```python
"""
Google Gemini Embedding 适配器
调用原生 API (非 OpenAI 兼容格式)，产出 embedding.npy
模型: gemini-embedding-001
  - 稳定版，纯文本，默认 3072 维
  - 推荐 task_type=RETRIEVAL_DOCUMENT（建库）/ RETRIEVAL_QUERY（检索）
  - 支持截断到 768/1536/3072 维

官方文档: https://ai.google.dev/gemini-api/docs/embeddings
"""
import os
import time
import numpy as np
import requests
from pathlib import Path
from typing import List
from dotenv import load_dotenv

load_dotenv()

GOOGLE_API_KEY = os.environ["GOOGLE_API_KEY"]
EMBED_MODEL    = "gemini-embedding-001"
EMBED_DIM      = 768          # 截断到 768 维，节省存储，质量接近 3072
EMBED_URL      = f"https://generativelanguage.googleapis.com/v1beta/models/{EMBED_MODEL}:batchEmbedContents"
HEADERS        = {"Content-Type": "application/json", "x-goog-api-key": GOOGLE_API_KEY}
BATCH_SIZE     = 20           # 原生 API 批量上限 ~100，保守取 20
RATE_DELAY     = 0.1          # 批次间延迟（秒），避免触发速率限制


def embed_texts(texts: List[str], task_type: str = "RETRIEVAL_DOCUMENT") -> np.ndarray:
    """
    批量将文本列表转换为向量矩阵。

    Args:
        texts: 文本列表
        task_type: "RETRIEVAL_DOCUMENT"（建库）或 "RETRIEVAL_QUERY"（查询）
    Returns:
        shape (N, EMBED_DIM) 的 float32 ndarray，已 L2 归一化
    """
    all_embeddings = []
    total = len(texts)

    for start in range(0, total, BATCH_SIZE):
        batch = texts[start : start + BATCH_SIZE]
        payload = {
            "requests": [
                {
                    "model": f"models/{EMBED_MODEL}",
                    "content": {"parts": [{"text": t}]},
                    "taskType": task_type,
                    "outputDimensionality": EMBED_DIM,
                }
                for t in batch
            ]
        }
        for attempt in range(3):
            resp = requests.post(EMBED_URL, json=payload, headers=HEADERS, timeout=30)
            if resp.status_code == 429:
                wait = 2 ** attempt * 2
                print(f"  速率限制，等待 {wait}s...")
                time.sleep(wait)
                continue
            resp.raise_for_status()
            break

        data = resp.json()
        for item in data["embeddings"]:
            vec = np.array(item["values"], dtype=np.float32)
            # L2 归一化，使内积等价于余弦相似度
            norm = np.linalg.norm(vec)
            if norm > 0:
                vec = vec / norm
            all_embeddings.append(vec)

        print(f"  已处理 {min(start + BATCH_SIZE, total)}/{total}")
        time.sleep(RATE_DELAY)

    return np.array(all_embeddings, dtype=np.float32)


def embed_query(query: str) -> np.ndarray:
    """单条查询向量（RETRIEVAL_QUERY task）"""
    return embed_texts([query], task_type="RETRIEVAL_QUERY")[0]


def build_embedding_npy(chunks_jsonl: str, output_npy: str) -> None:
    """
    读取 chunks.jsonl，对 contents 字段批量 embedding，保存为 .npy。
    供 agv_index 流程中替换 UltraRAG 的 retriever_embed 步骤。
    """
    import json
    rows = [json.loads(line) for line in Path(chunks_jsonl).read_text(encoding="utf-8").splitlines() if line.strip()]
    texts = [r.get("title", "") + "\n" + r.get("contents", "") for r in rows]

    print(f"开始 embedding {len(texts)} 条 chunks...")
    embeddings = embed_texts(texts, task_type="RETRIEVAL_DOCUMENT")

    Path(output_npy).parent.mkdir(parents=True, exist_ok=True)
    np.save(output_npy, embeddings)
    print(f"✅ 已保存: {output_npy}，shape={embeddings.shape}")


if __name__ == "__main__":
    # 直接运行时的快速测试
    vecs = embed_texts(["AGV 换电步骤", "电池规格参数"], task_type="RETRIEVAL_DOCUMENT")
    print(f"✅ 测试通过，维度: {vecs.shape}")
```
