> 将检索、Prompt 组装、vLLM 生成串联成完整链路，并通过一个最小 Flask API 端点进行端到端验证。

## 完整数据流

```
POST /api/chat
  {"kb_id": "agv_demo", "question": "换电步骤是什么"}
           │
           ▼
rag_runner.py: chat()
           │
           ├─ 1. retriever_search(question, top_k=8)
           │       ↓ 返回 ret_psg (含 contents / images / title)
           │
           ├─ 2. prompt 组装 (agv_qa_rag.jinja)
           │       ↓ 返回 filled_prompt
           │
           ├─ 3. generation.generate(filled_prompt)
           │       ↓ 调 vLLM gpt-oss-120b → answer 文字
           │
           └─ 4. 拼装响应
                   ├─ answer: 模型回答文字
                   └─ sources: [{title, snippet, images:[base64,...]}]
           │
           ▼
HTTP 200 JSON 响应
```

## 问答 Pipeline YAML

新建 `examples/agv_rag.yaml`：

```yaml
# AGV 知识库 — 问答 Pipeline
# 通过 rag_runner.py 调用，不直接 ultrarag run

servers:
  retriever:  servers/retriever
  prompt:     servers/prompt
  generation: servers/generation

pipeline:
- retriever.retriever_search:
    input:
      top_k: 8
- prompt.render:
    input:
      template_name: agv_qa_rag
- generation.generate
```

> **注意**：问答时 `retriever_init` 已在服务启动时完成（索引已加载），`retriever_search` 直接用内存中的索引查询，无需每次重新 init。

## `rag_runner.py` 核心逻辑（规划）

```python
# custom_app/services/rag_runner.py

import base64
from pathlib import Path
from typing import List, Dict, Any

class RagRunner:
    """
    封装 UltraRAG 问答流程：
      检索 → Prompt 组装 → vLLM 生成 → base64 图片拼装
    """

    def __init__(self, kb_id: str, kb_base_dir: str = "data/kb"):
        self.kb_id = kb_id
        self.kb_dir = Path(kb_base_dir) / kb_id
        self._retriever = None
        self._generator = None

    async def init(self):
        """启动时调用一次：加载索引到内存"""
        # 调用 UltraRAG retriever.retriever_init，加载 FAISS 索引
        # 参数从 servers/retriever/parameter.yaml 读取，路径动态替换 kb_id
        ...

    async def chat(self, question: str) -> Dict[str, Any]:
        """
        入参: question 字符串
        出参: {
          "answer": "...",
          "sources": [
            {
              "title":   "3.2 换电步骤",
              "snippet": "确认 AGV 已到达换电位...",
              "images":  ["data:image/png;base64,..."]
            }
          ]
        }
        """
        # Step 1: 检索
        ret_psg = await self._search(question)

        # Step 2: 组装 Prompt
        prompt = self._build_prompt(question, ret_psg)

        # Step 3: 调 vLLM 生成
        answer = await self._generate(prompt)

        # Step 4: 拼装 sources，图片转 base64
        sources = []
        for psg in ret_psg:
            images_b64 = []
            for img_rel_path in psg.get("images", []):
                img_abs = self.kb_dir / img_rel_path
                if img_abs.exists():
                    b64 = base64.b64encode(img_abs.read_bytes()).decode()
                    images_b64.append(f"data:image/png;base64,{b64}")
            sources.append({
                "title":   psg.get("title", ""),
                "snippet": psg.get("contents", "")[:200],
                "images":  images_b64,
            })

        return {"answer": answer, "sources": sources}

    def _build_prompt(self, question: str, passages: List[Dict]) -> str:
        """用 Jinja2 渲染 agv_qa_rag.jinja"""
        from jinja2 import Environment, FileSystemLoader
        env = Environment(loader=FileSystemLoader("prompt"))
        tmpl = env.get_template("agv_qa_rag.jinja")
        return tmpl.render(question=question, passages=passages, enumerate=enumerate)
```

## 最小 Flask API

新建 `custom_app/app.py`：

```python
# custom_app/app.py
"""Phase 1 最小验证 API"""

import asyncio
from flask import Flask, request, jsonify
from .services.rag_runner import RagRunner

app = Flask(__name__)
runner = RagRunner(kb_id="agv_demo")

@app.before_first_request
def startup():
    asyncio.run(runner.init())

@app.route("/api/chat", methods=["POST"])
def chat():
    data     = request.get_json()
    question = data.get("question", "").strip()
    if not question:
        return jsonify({"error": "question 不能为空"}), 400

    result = asyncio.run(runner.chat(question))
    return jsonify(result)

@app.route("/health")
def health():
    return jsonify({"status": "ok"})

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=8080, debug=False)
```

启动：

```bash
python -m custom_app.app
```

## 验收标准（T4）

1. `POST /api/chat` 返回 HTTP 200，`answer` 字段非空且与文档相关（非编造）。  
2. `sources` 至少包含 1 条，每条含 `title`、`snippet`。  
3. 问"换电步骤"类问题时，至少有 1 个 source 的 `images` 不为空数组。  
4. 将 `images[0]` 的值（`data:image/png;base64,...`）粘贴到浏览器地址栏，能显示对应图片。  
5. 问一个文档中没有的问题（如"AGV 价格是多少"），模型回答包含"文档中未找到相关内容"而非编造数据。

## Phase 1.5 — 回答质量优化（推荐实施）

> 说明：本节是在 T4 已跑通基础上的质量优化方案，优先复用 UltraRAG 原生能力，必要处借鉴 WeKnora 的工程做法。

### 现状问题与对应策略

- 回答偏啰嗦：先“高召回”再“强截断”，避免直接把大量片段喂给生成模型。
- 图文不匹配：将图片与 `source_id` 绑定，避免仅靠关键词匹配。
- 问题表述不规范导致检索偏移：增加 `rewrite_query` 预处理。

### 建议链路（最小可落地）

1. `rewrite_query(question) -> rewritten_question`  
2. `retriever_search(rewritten_question, top_k=12)`  
3. `rerank(rewritten_question, passages) -> top_3`  
4. `prompt_render(top_3, 强约束引用 source_id)`  
5. `generation(answer)`  
6. `answer_blocks` 按 `source_id` 绑定对应图片

### 参数建议（本机初级 GPU）

- `recall_top_k`: `12`
- `final_top_k`: `3`
- `rerank_batch_size`: `4`（显存紧张可降到 `2`）
- `generation_max_tokens`: 适度收紧，避免答案过长

### Prompt 约束建议

- 明确要求“只基于给定 sources 作答”。
- 每个关键结论后附 `source_id`。
- 缺失依据时明确输出“insufficient evidence”。
- 限制输出结构：步骤优先、避免重复解释。

### 与 WeKnora 对齐的可借鉴实现点（Phase 1 可用）

- QueryUnderstand（改写 + 意图分流）思路。
- FilterTopK 独立阶段，便于观察截断收益。
- Merge 去重与近重复清理，减少冗余上下文。
- Rerank 后再做多样性选择（可后续引入轻量 MMR）。

### 优化验收建议（对比基础版）

- 同一问题：`sources` 数量从 6~8 降到 3 左右。
- 回答字数下降，但关键步骤覆盖率不下降。
- 图片与答案步骤对齐率提升（人工抽样核验）。
- 端到端时延保持在可接受范围（记录 P50/P90）。
